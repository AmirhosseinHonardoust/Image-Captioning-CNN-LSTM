# Upgrade Summary

This patch converts the project from a clean educational baseline into a more reviewer-ready ML repository.

## Added

- GitHub Actions CI for `pytest` and `ruff check .`.
- `pyproject.toml` for pytest and Ruff configuration.
- `.gitignore` for Python caches, virtual environments, and generated model artifacts.
- `MODEL_CARD.md` documenting intended use, risks, evaluation, and limitations.
- `scripts/prepare_flickr_csv.py` for converting Flickr-style caption files into the project CSV format.
- Sample prediction export support: `outputs/sample_predictions.json` and `outputs/sample_predictions.csv` are generated after test evaluation.

## Improved

- BLEU evaluation now supports multiple reference captions per image.
- Evaluation groups repeated `image_path` rows as references for the same image.
- Training supports gradient clipping with `--grad-clip`.
- Training supports BLEU-4 early stopping with `--early-stopping-patience` and `--early-stopping-min-delta`.
- Metrics now include `best_epoch` and `num_samples`.
- README now clearly states that the checkpoint is generated by training and is not committed by default.
- README includes stronger custom dataset guidance and Flickr-style CSV conversion instructions.

## Verified

- Python syntax compilation passed for `src/*.py` and `scripts/*.py`.
- Local test suite passed: `6 passed`.

## Note

A full training smoke run was not completed in this sandbox because the preinstalled Torch/Torchvision packages are binary-incompatible here. The repository pins compatible versions in `requirements.txt` and the CI workflow uses Python 3.10 to avoid that environment issue.
